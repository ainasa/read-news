# Convenciones SDD

## Proceso

- Una feature por sesion.
- No mezclar cambios de varias features.
- No cerrar done sin review APPROVED.

## Documentacion

- Mantener specs claros y trazables.
- Registrar bloqueos en progress/current.md.
- Registrar implementacion en progress/impl_<feature>.md.
- Registrar review en progress/review_<feature>.md.

## Cambios

- Evitar cambios fuera del alcance del spec.
- Si el spec es insuficiente, volver a spec y corregir.
